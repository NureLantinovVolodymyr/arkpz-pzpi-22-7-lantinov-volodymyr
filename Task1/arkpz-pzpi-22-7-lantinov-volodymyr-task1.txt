1	МЕТА РОБОТИ

Метою даної лабораторної роботи є розробка документа Vision and Scope для програмної системи управління транспортними засобами. Система призначена для автоматизації управління автопарками, моніторингу технічного стану транспортних засобів, відстеження їх місцезнаходження в режимі реального часу, а також оптимізації маршрутів.

2	ХІД РОБОТИ

У рамках лабораторної роботи було проведено аналіз вимог до програмної системи та визначено її ключові функціональні аспекти. Основна увага приділялась розробці серверної частини системи, оскільки наразі веб- та мобільні клієнти не реалізовані. Було визначено, що система базується на інтеграції IoT-пристроїв, які збирають дані про транспортні засоби та передають їх на сервер. Серверна частина, реалізована за допомогою Node.js та Express, забезпечує обробку цих даних та їх зберігання у базі даних MongoDB.
Проведено також дослідження існуючих рішень на ринку, таких як Fleetio, Geotab та Wialon. Цей аналіз дозволив визначити недоліки аналогів, зокрема складність у використанні та відсутність інтеграції технічного моніторингу транспорту з GPS-відстеженням. На основі отриманих результатів сформовано унікальні переваги нашої системи, які включають централізоване управління автопарком, реальний моніторинг технічного стану та підтримку оптимізації маршрутів.
Поточна реалізація включає серверну частину системи, яка готова до інтеграції з IoT-пристроями. Веб-інтерфейс, хоча і планується до розробки, ще не впроваджено, тому доступ до функцій наразі можливий лише через API. Це рішення забезпечує масштабованість та підготовленість до подальшого розвитку системи.

3	ВИСНОВКИ

У результаті виконаної роботи було створено концепцію програмної системи для управління транспортними засобами, що базується на сучасних технологіях IoT та серверного програмного забезпечення. Головним досягненням є розробка серверної частини, яка забезпечує обробку даних з датчиків транспорту та їх зберігання в базі даних. Це є критичним компонентом системи, оскільки саме він забезпечує її функціональність.
Проведений аналіз аналогів підтвердив, що існуючі рішення не відповідають вимогам користувачів повною мірою, що створює значний потенціал для впровадження нашої системи. Інтеграція з IoT дозволяє автоматизувати процеси моніторингу, що значно скорочує витрати на обслуговування автопарку та покращує ефективність його роботи.
Подальші етапи роботи передбачають розробку веб- та мобільного інтерфейсів, які забезпечать зручний доступ до функцій системи для кінцевих користувачів. Це дозволить реалізувати повноцінний продукт, готовий до впровадження у реальних умовах.

1	BUSINESS REQUIREMENTS (БІЗНЕС-ВИМОГИ)

1.1	Background (Передумови)

AutoFleet Control — це проект, розроблений для ефективного управління автопарками та моніторингу стану транспортних засобів. Його основна мета — створити зручний інструмент для відстеження технічного стану автомобілів (двигун, температура, тиск у шинах) і місцезнаходження транспорту в реальному часі. Система допомагає підприємствам оптимізувати маршрути, знижувати експлуатаційні витрати та мінімізувати ризики, пов’язані з технічними несправностями.
Впровадження AutoFleet Control дозволяє компаніям підтримувати високу продуктивність автопарку за рахунок інтеграції IoT датчиків і GPS-трекерів. Користувачі системи отримують можливість зручно контролювати роботу транспорту через веб-інтерфейс або мобільний додаток, що забезпечує простий доступ до даних.

1.2	(Business Opprtunity) Бізнес можливості

AutoFleet Control — це інноваційна платформа для управління автопарками, яка забезпечує збирання та аналіз даних про транспортні засоби через інтеграцію IoT пристроїв. Основні функції включають моніторинг технічного стану транспорту, відстеження місцезнаходження, управління маршрутами та автоматичні нагадування про необхідність технічного обслуговування. Користувачі отримують персоналізовані сповіщення, що дозволяє їм своєчасно вирішувати потенційні проблеми.
Серед основних переваг платформи:
•	Інтеграція з IoT датчиками та GPS-трекерами для повного моніторингу.
•	Можливість створення ефективних маршрутів, що знижує витрати на пальне.
•	Простий у використанні веб-інтерфейс для адміністрування.
Проте існують деякі виклики, зокрема залежність від стабільного інтернет-з’єднання та потенційні складнощі з інтеграцією несертифікованих пристроїв. У порівнянні з конкурентами, такими як Fleetio або Geotab, AutoFleet Control пропонує більш збалансований підхід до технічного моніторингу і GPS-трекінгу в межах одного рішення, що знижує загальну вартість володіння системою.

1.3	Способи монетизації

•	Підписка для користувачів: Система може пропонувати базовий безкоштовний функціонал із можливістю переходу на преміум-план. Такий план включатиме розширені функції, наприклад, автоматизовані звіти про стан транспортних засобів, доступ до історичних даних, прогнозування технічних несправностей або аналітичні інструменти для оптимізації маршрутів.
•	Партнерства з логістичними компаніями: Надання інтеграції з їхніми системами управління замовленнями та створення індивідуальних рішень під ключ. Також можлива реклама послуг партнерських компаній (наприклад, страхових чи ремонтних).
•	Реклама: Розміщення контекстної реклами партнерів, таких як виробники GPS-трекерів, датчиків або технічних сервісів, у веб-інтерфейсі або мобільному додатку.
•	Консультаційні послуги: Надання послуг з впровадження системи, навчання персоналу користувача та кастомізації функціоналу під потреби великих компаній.
•	Додаткові послуги: Пропонування розширеної аналітики або API-доступу для інтеграції з іншими бізнес-процесами клієнтів за окрему плату.
 
2	VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)

2.1	Vision Statement (Окреслення концепції)

Програмна система для управління транспортними засобами є інтелектуальним інструментом для ефективного моніторингу та управління автопарком. Вона забезпечує можливість відстеження місцезнаходження транспортних засобів в реальному часі за допомогою GPS-трекерів, а також контролює стан транспортних засобів, використовуючи датчики для моніторингу стану двигуна, температури та тиску в шинах. Система також надає можливість керувати користувачами та транспортними засобами, призначати маршрути, а також відслідковувати статус технічного обслуговування та ефективність роботи транспорту.
Ця система дозволяє користувачам (менеджерам автопарку, водіям та логістичним компаніям) оперативно отримувати інформацію про стан транспорту та оптимізувати процеси управління автопарком, що призводить до зменшення витрат, підвищення безпеки та ефективності.

2.2	Major Features (Головна функціональність)

Функції, що будуть виконуватися системою:
MF-1) Відстеження місцезнаходження транспортних засобів — система забезпечує реальне відображення місцезнаходження транспортних засобів за допомогою GPS-трекерів, що дозволяє здійснювати моніторинг в реальному часі.
MF-2) Моніторинг стану транспортних засобів — система збирає та аналізує дані з датчиків для моніторингу стану двигуна, температури та тиску в шинах, що дозволяє своєчасно виявляти проблеми і запобігати несправностям.
MF-3) Управління користувачами та транспортними засобами — можливість додавання, редагування та видалення користувачів і транспортних засобів, а також призначення транспортних засобів користувачам і відключення транспортних засобів від користувачів.
MF-4) Планування маршрутів — система дозволяє створювати і видаляти маршрути для транспортних засобів, що полегшує організацію ефективних перевезень.
MF-5) Оновлення статусу транспортних засобів — можливість змінювати статус транспортного засобу (активний/неактивний), що дозволяє вчасно реагувати на зміни в експлуатації транспорту.
MF-6) Інтеграція з іншими пристроями та системами — підтримка синхронізації з іншими пристроями, такими як навігаційні системи та додатки для управління логістикою, для поліпшення загальної ефективності роботи системи.


3	SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ ПРОЄКТУ)

3.1	Scope of Initial Release (Рамки первинного випуску)

Перелік функціональності для серверної, веб та IoT частини програмної системи для управління транспортними засобами.
•	MF 1-1) Збір та відображення місцезнаходження транспортних засобів в реальному часі через GPS-трекери.
•	MF 1-2) Збір та відображення даних з датчиків для моніторингу стану двигуна, температури та тиску в шинах транспортних засобів.
•	MF 2-1) Можливість додавання, редагування та видалення транспортних засобів і користувачів системи.
•	MF 3-1) Інтеграція з основними GPS-трекерами та датчиками для збору даних про транспортні засоби.
•	MF 3-2) Можливість налаштування профілю транспортного засобу для точного відображення історії та стану.
•	MF 4-1) Захист персональних та технічних даних згідно з міжнародними стандартами безпеки.
•	MF 5-1) Панель адміністратора для моніторингу активності транспортних засобів та управління акаунтами користувачів.
•	MF 6-1) Веб-платформа для зручного доступу до даних, з можливістю інтеграції з хмарними сервісами для надійного зберігання інформації.

3.2	Scope of Subsequent Releases (Рамки наступних випусків)

Перелік функціональності для наступних версій програмної системи для управління транспортними засобами.
•	MF 1-3) Додавання аналізу ефективності використання транспорту, включаючи моніторинг витрат палива та пробігу.
•	MF 2-2) Розширення інтеграції з додатковими датчиками та пристроями для покращення збору даних (наприклад, температурні сенсори).
•	MF 3-3) Розширення можливості для автоматичного планування маршруту на основі технічного стану транспортного засобу та дорожніх умов.
•	MF 4-2) Покращення персоналізації системи з урахуванням додаткових індивідуальних характеристик транспортних засобів і користувачів.
•	MF 5-2) Оптимізація функцій адміністрування даних, включаючи можливість редагування профілів транспортних засобів та користувачів.
•	MF 6-2) Локалізація інтерфейсу для підтримки основних мов і регіонів, що дозволить адаптувати систему для міжнародного використання.

4	BUSINESS CONTEXT (БІЗНЕС-КОНТЕКСТ)

4.3	Operating Enviroment (Робоче середовище)

Система буде використовуватися користувачами в різних регіонах і часових зонах, забезпечуючи доступ до даних про стан транспортних засобів в будь-який час доби. Основне завдання системи — забезпечити точне і своєчасне відображення інформації про місцезнаходження транспортних засобів, стан їхнього технічного оснащення (двигун, шини, температура) та історію використання. Користувачі зможуть переглядати дані про транспортні засоби в реальному часі, отримувати сповіщення про необхідність профілактичного обслуговування чи технічних перевірок.
Система передбачає інтеграцію з різними типами пристроїв для збору даних: GPS-трекерами, датчиками для моніторингу стану двигуна, температури та тиску в шинах, а також підтримку мобільних додатків та веб-платформи для зручного перегляду та управління інформацією. Це вимагає стабільного інтернет-з'єднання та високої надійності для безперебійної роботи системи та коректного надання актуальних сповіщень і рекомендацій.
Мови програмування:
•	JavaScript (Node.js, Express): Використовуватиметься для розробки серверної частини, яка забезпечує ефективну роботу RESTful API та інтеграцію з базою даних для зберігання інформації про транспортні засоби та користувачів.
•	JavaScript (React): Використовуватиметься для створення інтерактивного веб-інтерфейсу, що дозволяє користувачам зручно взаємодіяти з системою.
Інфраструктура серверної частини:
•	Express: Веб-фреймворк для створення RESTful API та організації серверної логіки.
•	MongoDB: NoSQL база даних для зберігання даних про користувачів, транспортні засоби та їхню історію.
•	Mongoose: ORM для MongoDB, що полегшує роботу з базою даних.
•	MQTT: Протокол для обміну повідомленнями між IoT пристроями (GPS-трекерами, датчиками) і сервером для збору даних в реальному часі.
•	JWT: Система авторизації на основі токенів для забезпечення безпеки даних користувачів та доступу до них.
•	Docker: Контейнеризація серверної частини для полегшення процесів розробки, тестування та деплойменту.
Інфраструктура фронтенд частини:
•	React: Бібліотека для створення веб-інтерфейсу, що дозволяє користувачам зручно переглядати дані про свої транспортні засоби.
•	Material-UI: Для стилізації веб-інтерфейсу, щоб забезпечити привабливий та зручний дизайн.
•	Axios: Бібліотека для виконання HTTP-запитів до серверу та взаємодії з API для отримання даних про транспортні засоби.
Інфраструктура мобільного додатку:
•	Kotlin/Java (Android): Для розробки мобільного додатку, який буде збирати дані з GPS-трекерів і датчиків, а також передавати їх на сервер для подальшої обробки та зберігання.
